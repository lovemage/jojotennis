02球具評測（球具部落格）功能

05/27 會議決定：原「社團 / 討論區」流量起不來，改為由站方或編輯定期發佈的球具評測內容頁，會員不需貼文，類似一個品牌部落格，對 SEO 與會員留存友善。

內容類型與欄位

欄位說明
分類球拍 / 網球線 / 球鞋 / 握把（後台可調整）
標題、品牌、型號例：「Wilson Pro Staff v15 — 進階控制型球拍」
封面圖產品照或開箱情境，自動透過 Cloudinary 派送多尺寸
本文支援圖文混排（Markdown），內文配圖同樣走 Cloudinary
狀態草稿 / 已發佈，可排程發佈
網址每篇獨立網址（例：/reviews/wilson-pro-staff-v15），SEO 友善

Sabrina（或編輯）能做什麼

  • 進 /admin/reviews 後台撰寫、編輯、預覽、發佈文章。
  • 上傳封面與內文配圖（含拖放上傳、自動壓縮、多檔批次）。
  • 設定分類、品牌、型號，方便會員從前台分類過濾。
  • 排程未來時間發佈（適合內容檔期管理）。

會員能做什麼

  • 瀏覽 /reviews 列表頁，可依分類（拍/線/鞋/握把）過濾。
  • 閱讀單篇詳細頁，支援收藏、社群分享。
  • 本期不開放會員自行貼文，避免討論區內容失控；未來如需開放可另行擴充。

03約打評論系統｜會員信譽機制

每個約打日期過後，host 與 player 互相評分，建立「參加率」與「履約率」指標。避免放鳥行為、提升 community 品質，是這次改造的核心信任建立機制。

雙向評分流程

Host 評 Player

  • 是否出席：有到 / 沒到 / 請假（三選一）
  • 星等：1–5 星
  • 可選評論文字

Player 評 Host

  • 星等：1–5 星
  • 可選評論文字
  • （無「是否出席」欄位）

頭像旁顯示的兩個指標

指標對象公式概念
參加率 所有會員（以 player 身份） 有到場次 ÷ 應到場次，採「貝氏平滑」公式，新會員預設 75%
履約率 Host（開團者） 如期舉辦場次 ÷ 開團總場次（host 自己取消會扣履約率）

公平機制（避免誤傷）

  • 未滿 5 場：顯示「新會員」徽章，不顯示百分比，避免低樣本誤判。
  • Host 7 天沒評論 → 自動視為「有到」，不冤枉沒被評到的會員。
  • Player 48 小時前退出 → 不計缺席，給合理彈性（但 48 小時內退出算缺席）。
  • Host 取消整場 → 不影響 player 參加率（但會計入 host 的履約率）。
  • 請假機制：host 評論時可勾「正當理由請假」，該場不列入 player 分母。

隱私與糾紛預防

  • 個別評論文字預設不公開，只有本人與管理員可見。
  • 點對方頭像 → 只看得到「參加率 / 履約率 + 平均星評」聚合分數，看不到誰寫了什麼。
  • 同一場約打、同一對 reviewer-reviewee 只能評一次，避免重複刷分。
  • 連續多筆極端評分（全 1 星 / 全 5 星）會被系統標記給管理員，本期紀錄為主、不自動扣分。

04通知怎麼送達會員

推播與 Email 兩條通道並行，確保會員不論有沒有安裝 PWA、有沒有同意通知權限，都不會錯過重要訊息。

通道 A · PWA 推播

會員「加到主畫面」安裝 PWA，並同意通知權限後 → 約打申請 / 被接受 / 聊天訊息以手機系統推播送達。

通道 B · Email（Fallback）

會員沒安裝 PWA、或未同意推播權限 → 約打申請 / 被接受 / 歡迎信 / 站方公告仍以 Email 送達，不漏訊息。

事件對照表

事件推播（PWA）Email
新會員歡迎—✓
約打申請（host 收）✓✓
約打被接受 / 婉拒（player 收）✓✓
即時聊天訊息✓—
站方批量公告—✓

重要事件（約打申請、被接受/婉拒）雙通道同時發送；聊天訊息只走推播避免 Email 噪音；歡迎信與公告只走 Email。

05您的數位資產清單半自助

依照半自助開發服務模式，以下雲端帳號由 Sabrina 直接註冊與持有，工作室提供圖文教學引導，並在收到 API key 後執行所有開發整合。每項申請約 5–15 分鐘。

⚠ 申請時請用 Email / Password 註冊，不要用 Google 一鍵登入

Supabase、Resend 等服務都有「用 Google 登入」的快速申請選項，請不要選，請改用 Email / Password 註冊。
原因：Google 一鍵綁定後，帳號控制權會綁在 Google 帳號上，未來轉交、團隊共用、或更換開發團隊時都會卡關。Email / Password 註冊比較單純，帳號就是帳號，符合「資產完全歸您所有」原則。

09驗收清單

完工後工作室提供 Preview 連結（不影響正式網站），Sabrina 依以下清單實際操作驗證：

  1. 後台新增 / 編輯一個球場 → 前台地圖與圖片輪播顯示正常。
  2. 用 LINE 登入註冊新會員 → 收到歡迎信。
  3. 開一場約打 + 另一帳號申請 → host 收推播 + Email；接受後 player 也收。
  4. 手機（Android + iPhone）加主畫面開啟為 App 模式並收到推播。
  5. 約打日後 host 評論 player → 雙方頭像旁顯示參加率。
  6. 後台批量公告功能 → 寄信給選定的會員群組。

通過後即正式上線到 jojotennis.com。
